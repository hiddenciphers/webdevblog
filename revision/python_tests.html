Performance Testing

  • Performance testing assesses the efficiency of a piece of code or an entire application.

  • It evaluates factors such as speed, responsiveness, & scalability.

  • The size of the code being tested can range from a function to the entire application.

  • Performance testing helps identify bottlenecks, optimise code, & ensure the application meets performance expectations under various conditions, such as heavy load or concurrent users.



Unit Testing

  • Unit testing focuses on the smallest parts of the software design, typically individual functions or modules.

  • It checks if the actual results of these units match the expected results.

  • Unit tests can be conducted manually or automatically through testing frameworks.

  • Unit testing ensures that each component of the software functions correctly in isolation.

  • It aids in early detection of bugs and facilitates code maintenance & refactoring.



End-To-End Testing

  • End-to-end testing involves testing the entire application, simulating real-world scenarios.

  • The goal is to guarantee the stability of the application, ensuring that all components work together seamlessly.

  • These tests should still pass with minimal effort when new code is added.

  • End-to-end testing validates the flow of data & interactions between different modules of the application, helping to identify any integration issues.

  • It provides a comprehensive view of the system's functionality.



Acceptance Testing

  • Acceptance testing is the final phase of testing that assesses whether the application is ready for deployment.

  • It checks the application against the client/business requirements to confirm that it meets their needs & is aligned with the specified criteria.

  • Acceptance testing provides assurance that the developed software satisfies the client's expectations & is ready for release.

  • It is crucial for ensuring the overall quality & reliability of the application.
